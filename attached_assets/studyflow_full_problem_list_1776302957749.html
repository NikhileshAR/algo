StudyFlow — complete problem list comparing original vision to current build

Missing from vision UX/feedback gaps Data/intelligence gaps Product completeness
The big one — the entire original concept is missing
1
No browser extension — the passive tracking never got built
Fix: Build a Chrome/Firefox extension that logs domain + time spent. POST to /api/sessions automatically when a study-related domain is visited for 2+ min. The whole point was zero manual logging.
2
No automatic site-to-topic mapping
Fix: Maintain a mapping: "khanacademy.org/math" → topic id 4. Let users manually tag unknown domains. Extension sends domain → backend resolves topic → auto-logs session.
3
No distraction detection — off-task browsing is invisible
Fix: Track active vs background tab state. If user has YouTube open alongside Khan Academy, flag it. The discipline score (D) needs to know when they drifted.
4
No extension popup / browser-native UI
Fix: Extension toolbar icon shows today's next scheduled block. One-click to start a timed session. No need to open the web app at all during study.
UX and feedback gaps
5
Logging a session feels like nothing happened
Fix: After saving, show "Mastery: 38% → 51%" in the toast. Animate the progress bar on the topic card. Make the update visible, not silent.
6
The algorithm never speaks — numbers sit silent
Fix: Add an insight card: "Discipline dropped to 61% — today's schedule is lighter. Last studied Chemistry 9 days ago — it's urgent." Plain English from the state vector.
7
Onboarding drops you in an empty dashboard with no guidance
Fix: Show a 3-step checklist: ☐ Add your first 3 topics → ☐ Run schedule → ☐ Log first session. Dismiss when complete. System is useless until these happen.
8
Schedule blocks have no context — just a topic name and a timer
Fix: Under each block: "Last studied 11 days ago · 43% mastery · prerequisites met · focus on understanding, not practice yet."
9
No in-app study timer — you have to guess your duration
Fix: "Start session" button begins a timer. When you stop, it pre-fills the duration in the log form. Actual time, not estimated.
10
Topics have no connection to exam outcome
Fix: Add a "Readiness forecast" bar above the topic list. Project coverage at exam date based on current mastery velocity. "At this pace: 68% coverage by May 12."
Data and intelligence gaps
11
Discipline score (D) is based on self-reported data — easily gamed
Fix: Once the extension exists, D should use verified active-tab time, not what the user typed. Self-reported hours are useless for a discipline metric.
12
No spaced repetition — topics don't resurface at optimal intervals
Fix: Add lastStudiedAt decay to the priority function. A topic mastered at 80% but not touched in 14 days should resurface — memory fades. SM-2 or a simplified forgetting curve.
13
No per-topic learning velocity tracking
Fix: Track mastery delta per session per topic. Surface: "Chemistry gains 8% per session — Math only 2%. Allocate more time to Math." The scheduler should factor in how quickly topics actually improve.
14
Prerequisite logic is silent — users can't see why topics are blocked
Fix: On blocked topics, show: "Waiting on Calculus I (currently 44% — needs 60%)". Make the dependency graph visible, not just operative.
15
No time-of-day awareness — schedule ignores when you study best
Fix: Track mastery gains by hour-of-day across sessions. Surface: "Your best mastery gains happen 8–10am. Hard topics scheduled then by default."
16
Confidence score (c_i) exists in the schema but is never updated or displayed
Fix: When a practice session is logged with test scores, update confidenceScore. Show it separately from masteryScore — you can have high mastery but low confidence if you haven't attempted many questions.
Product completeness
17
No reminders — the app only works if you remember to open it
Fix: Browser notifications (via extension) or email reminders at scheduled session times. "Your Organic Chemistry block starts in 15 min."
18
Mobile layout is functional but not great — spacing and tap targets are off
Fix: Audit touch targets on the schedule page. The "Done" buttons need to be 44px minimum. Sidebar collapses well but the hamburger trigger is too small.
19
No per-topic history — you can't see how mastery changed over time
Fix: Click a topic → see a small sparkline of mastery over sessions. Shows whether the formula is actually working or if a topic keeps fluctuating.
20
Adding topics manually one by one is painful for real exams
Fix: Add CSV import for topics. USMLE Step 1 has 300+ topics — no one is entering those by hand. Even a paste-a-list text box would help massively.
21
No weekly review — the system never pauses to reflect
Fix: A Sunday summary view: sessions logged, mastery gained per subject, topics that got skipped, one recommendation for the coming week. Closes the weekly loop.
